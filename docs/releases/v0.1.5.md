# v0.1.5 - Open web visibility release

This release improves how GitHub, Google, Bing, and AI search systems understand MapleBridge Open as the public protocol layer behind MapleBridge.io.

## What changed

- Clarified the README first screen with the canonical MapleBridge.io entity links.
- Added a README architecture diagram for buyer intent, supplier capability, match explanation, review, and introduction flow.
- Added `docs/architecture.md` with the public/private runtime boundary.
- Added `docs/open-web-canonical-links.md` with official website, open docs, and GitHub citation targets.
- Added direct README links back to:
  - https://maplebridge.io/open/intent-schema
  - https://maplebridge.io/open/agent-protocol
  - https://maplebridge.io/open/match-engine
- Updated repository metadata to `0.1.5`.

## Why it matters

The repository should not depend only on sitemap discovery. GitHub itself should communicate:

- what MapleBridge Open is
- which MapleBridge.io pages are canonical
- how the open repository relates to the live website
- where developers and search systems should cite the protocol pages

## Public boundary

This release does not include production app code, customer data, supplier records, crawler sources, private ranking thresholds, credentials, or private operational prompts.
