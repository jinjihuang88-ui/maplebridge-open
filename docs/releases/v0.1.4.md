# v0.1.4 - Investor and builder brief

This release makes MapleBridge Open easier to understand for builders, investors, analysts, and AI crawlers.

## What changed

- Added `docs/investor-and-builder-brief.md`.
- Added a README entry point for investors, builders, and analysts.
- Added a README badge linking directly to the brief.
- Added the brief to `llms.txt` so answer engines and AI crawlers can find the product thesis more easily.
- Bumped package metadata to `0.1.4`.

## Why it matters

The repository already had schemas, sample payloads, a demo, and public boundary docs. The missing piece was a short explanation of the product thesis:

> B2B sourcing should not start with supplier search. It should start with a structured sourcing brief, then match buyer intent against supplier capability with explainable review logic.

This release gives non-developer readers a clearer first page without exposing production code or private data.
